# Supabase — DESIGN.md

**Personality:** Dark, developer-native, confident. The "open-source Firebase" look — near-black surfaces, a signature emerald green, monospace touches, and a terminal-adjacent feel that engineers trust instantly.

## Color
- Brand: Supabase green — `#3ECF8E` (emerald). The single, unmistakable accent.
- Canvas: near-black — `#1C1C1C` / `#171717` / `#0F1115`. Dark-first.
- Text: off-white `#EDEDED` + muted gray `#A0A0A0`.
- **Take-away:** dark canvas + emerald `#3ECF8E` + restrained neutrals = developer-native.

## Typography
- UI/body: clean sans (Inter-like / "Custom" / system).
- Mono: a monospace face for code, keys, and technical labels — used liberally (the brand *is* technical).
- Crisp hierarchy; code samples are first-class citizens.

## Layout & spacing
- Dense-but-clean; strong grid; code blocks and terminal motifs throughout.
- Small radii (~6–8px); hairline borders; subtle green glows behind heroes.

## Components
- Buttons: emerald primary, dark label; ghost/bordered secondary. Crisp, fast.
- Cards: dark surface, hairline border, mono labels, code-forward. Tight padding.
- Signature: green-on-dark, SQL/code snippets, the elephant mark, terminal aesthetics.

## Motion
- Fast, subtle, functional. Crisp hovers; occasional green glow. Engineered, not flashy.

## Voice (microcopy)
- Direct, technical, open-source-community. Builder-to-builder, transparent.

## Signature moves
- Near-black + emerald `#3ECF8E` + monospace + code-forward layout = "Supabase."

## When to reference this
Briefs wanting **dark, developer-native, technical, open-source** — dev tools, APIs, databases, infrastructure, anything that should earn engineer trust at a glance.
