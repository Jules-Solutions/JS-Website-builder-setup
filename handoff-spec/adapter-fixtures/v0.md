> Per-tool handoff fixture for **v0** (v0.app / v0.dev by Vercel). React/Next-flavored output that auto-handles shadcn — the brief's `output_format` library=`shadcn` maps cleanly. Canonical design anchor: `DESIGN-handoff-protocol.md` § "Adapter fixtures". Brief contract: `handoff-spec/component-request-v1.md`. Output contract: `handoff-spec/component-output-v1.md`.
>
> Tool behaviour verified 2026-06-14 (see "Sources"). Re-verify on a cadence — v0 ships fast (Feb 2026 added Git integration, a VS Code-style editor, DB connectivity, agentic workflows).

## How to paste the brief into v0

1. Open v0 (v0.app). Start a **new generation**.
2. Paste the **JSON brief** as the prompt. v0 reads the `request` + `output_format` and plans a component tree mapped onto shadcn primitives.
3. v0 **streams TypeScript + JSX into a live preview pane** while styles compile in a sandboxed Next.js runtime. Watch the preview match the brief's intent.
4. If the result drifts from the brand, iterate in v0's chat ("use the brand primary token, not indigo") — but the cleaner path is to capture, ingest, and let round 2's `iteration_history` drive the fix.

## Tool-specific quirks

- **Next + shadcn + Radix + Tailwind by default.** v0's house stack is exactly the brief's common `output_format` (react / shadcn / tailwind / tsx) — so the mapping is the cleanest of all the AI tools. `library: shadcn` is honored natively.
- **Real, runnable TS — not a mockup.** Output is real React with imports, props, and state. It ships consistent spacing tokens and a proper compound-component hierarchy, with zero lint warnings against a default shadcn config.
- **It reaches for shadcn's default theme palette.** v0 uses shadcn's CSS-variable theme (`bg-primary`, `text-foreground`, etc.) by default. That's *good* — those are token references, not hardcoded hex — but the variables resolve to shadcn's defaults unless the project's `globals.css` maps them to the brand OKLCH tokens. On ingest, confirm the CSS-variable theme is wired to `brand.yaml` tokens; flag any raw `indigo-*`/`slate-*` that slipped in.
- **Multi-file leanings.** v0 may split a component into a few files (component + a `lib/utils` import, a sub-component). When `file_count_hint: 1`, it usually still respects a single primary file; for richer components set `file_count_hint > 1`.
- **Copy or download.** Output can be copied from the code panel or downloaded; Feb-2026 v0 can also push to a connected Git repo.

## Expected output format

Real React/TS (`.tsx`), shadcn-flavored, usually **Form 1 (pure code)** per `component-output-v1.md`. v0 does not natively emit the Form-2 metadata header; capture as Form 1.

## How to capture output

- **Copy** from v0's code panel, or **Download** the file(s).
- Paste back to your Claude Code agent, or save to `.website-builder/outputs/{brief-id}.tsx`.
- If v0 produced multiple files, save each; the parser splits `// File:` markers (Form 3) or you save them individually under the brief id.

## Known issues

- **shadcn theme variables resolve to defaults** unless `globals.css` is brand-mapped — looks on-brand in v0's preview, drifts in your project if the theme isn't wired. Ingest checks the CSS-variable wiring.
- **Occasional raw Tailwind palette leak** (`bg-indigo-500`) — palette validator flags; round-2 brief lists it.
- **Extra sub-components** beyond the brief — prop/shape-deviation check surfaces them for your decision.
